The problem

On-chain order books are gas-bound.

A conventional on-chain CLOB runs matching, mark-to-market, funding and liquidations inside a transaction — so throughput is capped by the block gas limit. That's why every serious order-book perp lives off-chain:

  • dYdX → a Cosmos app-chain (off-chain matching)
  • Lighter → a ZK-rollup (validity proofs)
  • Vertex → an off-chain sequencer (on-chain re-check)
The idea

Store a commitment, not the exchange.

Gas Killer inverts a normal contract. Instead of one storage slot per order and per position, it stores only keccak256(state). The expanded state is handed to the contract as a witness on every call and verified by re-hashing.

The book isn't on-chain, so reads and writes of it cost nothing on-chain — the exchange's size is unbounded while its on-chain footprint is fixed at one slot.

The commitment

The whole exchange, one 32-byte word.

# one storage slot commits everything STATE_SLOT = { bits[255:64] = uint192( keccak256(canonical_state) ), bits[63:0] = uint64( lastTransitionAt ) } # every mutator verifies the witness, then re-commits require( keccak256(witness) == committed_hash ) // StateWitnessMismatch

Accounts, cross-margin positions, resting orders, oracle prices, funding indices, the insurance fund — all of it hashes into that one slot.

How Gas Killer enables it

Compute off-chain. Commit one write.

1
State is a commitment, not a layout. The book's size is unbounded; the on-chain footprint is one slot.
2
Unbounded off-chain execution. Operators run settleEpoch under SimProfile::UnboundedV1 (2⁴⁰ gas) — a 1,200-order match costs ~527M gas, ~17× a block.
3
The result is a bounded diff. However much it computes, the on-chain effect is a single STORE(STATE_SLOT, newCommitment).
4
Restaked operators attest. A ≥66% EigenLayer BLS quorum applies the STORE via verifyAndUpdate. Economic trust — not a validity proof — and honestly labelled so.
5
A liveness backstop. A 7-day inactivity gate lets any user exit even if operators go dark.
The honest architecture

Solidity is the vault + verifier — not the exchange.

"Everything in Solidity" would be the gas-bound thing we're escaping. The real deal is a hybrid, and it's the deal that buys unbounded compute:

On-chain
Custody of collateral · the 32-byte commitment · witness & quorum verification · replay guard · the emergency exit.
Off-chain
The full state · matching · funding · liquidation math · the operator that holds it all and serves the witness.
The trade
Move state & compute off-chain → get unbounded throughput and O(1) on-chain state. You can't have that and "all on-chain."
The off-chain state store

One honest operator is enough.

Every mutating call carries its semantic action in calldata — which is on-chain data. So the canonical state is reconstructable by anyone from the chain. The operator can serve the truth or censor, but it can never forge (the contract rejects a wrong witness).

  • server/ — a Node operator: rebuilds state from calldata, self-heals via event replay, serves the witness (GET /state · POST /apply · GET /health)
  • reconstruct.ts — the same logic in the browser, so the static site is self-contained: the browser is its own read-only operator
